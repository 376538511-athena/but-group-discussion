# 组会文献讨论平台 — 功能框架文档

## 一、项目概述

**项目名称**：研究生组会文献讨论平台
**目标用户**：研究生组会成员（30+ 人）
**核心定位**：以"强制评论追踪"为特色的学术文献协同讨论工具
**技术架构**：React + Node.js + PostgreSQL 前后端分离 SPA

---

## 二、用户角色

| 角色 | 权限 |
|------|------|
| **管理员 (Admin)** | 全部功能 + 成员管理（启用/停用/角色变更）+ 统计导出 |
| **成员 (Member)** | 上传文献、评论、点赞、查看统计、修改个人资料 |

---

## 三、功能模块

### 模块 1：用户认证系统

| 功能点 | 说明 |
|--------|------|
| 注册 | 填写姓名、用户名、邮箱、密码、学号（选填）、研究方向（选填） |
| 登录 | 用户名 + 密码，JWT 令牌认证 |
| 令牌刷新 | Access Token 2 小时过期，Refresh Token 7 天自动续期 |
| 修改密码 | 验证旧密码后设置新密码 |
| 修改资料 | 修改姓名、邮箱、学号、研究方向 |

### 模块 2：文献管理

| 功能点 | 说明 |
|--------|------|
| 上传文献 | 填写标题、作者、摘要、演讲日期，上传 PDF（限 50MB） |
| 文献列表 | Google Scholar 风格卡片列表，分页展示 |
| 搜索筛选 | 支持按标题/作者/摘要关键词搜索，按上传时间/标题排序 |
| 文献详情 | 完整元数据展示：标题、作者、摘要、上传者、上传日期、演讲日期 |
| PDF 下载 | 在线下载原始 PDF 文件 |
| 编辑/删除 | 上传者或管理员可编辑元数据、删除文献 |

**文献卡片信息结构**：
```
[标题] — 蓝色链接样式
[作者列表] — 绿色
[摘要预览（2行截断）] — 灰色
[上传者: xxx | 日期: yyyy-mm-dd | N 评论 · M 人参与]
[状态徽章: 待完成 Pending / 已参与 Engaged / 我上传的]
```

### 模块 3：评论讨论系统

| 功能点 | 说明 |
|--------|------|
| 发表评论 | 在文献下发表顶级评论（文本） |
| 回复评论 | 对已有评论进行回复，最多支持 3 层嵌套 |
| 编辑评论 | 编辑自己发表的评论 |
| 删除评论 | 删除自己的评论（管理员可删除任意评论） |
| 点赞互动 | 对任意评论点赞/取消点赞，显示点赞计数 |

**讨论区 UI 风格**：GitHub Issue 式线程布局
- 左侧头像 + 用户名 + 时间戳
- 评论正文
- 操作栏：👍 点赞数 | 回复
- 嵌套回复通过左侧竖线缩进展示

### 模块 4：强制评论追踪（核心特性）

| 功能点 | 说明 |
|--------|------|
| 参与状态徽章 | 文献卡片右侧显示当前用户状态：红色"待完成 Pending" / 绿色"已参与 Engaged" |
| 左侧边框标识 | 文献列表左侧彩色边框：红 = 待评论 / 绿 = 已评论 / 蓝 = 我上传的 |
| 人员追踪面板 | 文献详情页展示"已参与"和"待完成"两栏成员名单 |
| 上传者免评 | 文献上传者自动免除评论义务 |
| 待评论提醒 | 仪表盘首页高亮显示当前用户所有待评论文献 |

**追踪规则**：
- 每篇文献上传后，除上传者外的所有活跃成员均需评论
- 任何一条顶级评论或回复即视为"已参与"
- 未评论人员实名公示在文献详情页

### 模块 5：统计面板

| 功能点 | 说明 |
|--------|------|
| 仪表盘 | 4 张统计卡片：总文献数、我的评论数、待评论数、组会成员数 |
| 参与矩阵 | 全员 × 全文献的交叉表格，单元格显示：✓ 已评论 / ✗ 未评论 / 上传者 |
| 个人统计 | 查看任意成员的参与历史 |
| 数据导出 | 管理员可导出参与统计为 CSV（供导师查阅） |

### 模块 6：成员管理（管理员）

| 功能点 | 说明 |
|--------|------|
| 成员列表 | 表格展示全部注册成员：姓名、用户名、学号、邮箱、研究方向 |
| 角色变更 | 设置成员为管理员或普通成员 |
| 启用/停用 | 停用账号后该成员无法登录，不计入评论追踪 |

## 六、API 接口清单

### 认证 /api/v1/auth
- `POST /register` — 注册
- `POST /login` — 登录（返回 JWT）
- `POST /refresh` — 刷新令牌
- `GET /me` — 获取当前用户信息
- `PUT /password` — 修改密码

### 文献 /api/v1/papers
- `GET /` — 文献列表（分页/搜索/筛选）
- `GET /:id` — 文献详情（含参与状态）
- `POST /` — 上传文献（multipart/form-data）
- `PUT /:id` — 编辑文献元数据
- `DELETE /:id` — 删除文献
- `GET /:id/download` — 下载 PDF

### 评论 /api/v1
- `GET /papers/:id/comments` — 获取评论列表（线程化）
- `POST /papers/:id/comments` — 发表评论/回复
- `PUT /comments/:id` — 编辑评论
- `DELETE /comments/:id` — 删除评论
- `POST /comments/:id/like` — 点赞/取消点赞

### 用户 /api/v1/users
- `GET /` — 成员列表（管理员）
- `GET /:id` — 成员详情
- `PUT /:id` — 编辑资料
- `PUT /:id/role` — 变更角色（管理员）
- `PUT /:id/status` — 启用/停用（管理员）

### 统计 /api/v1/stats
- `GET /overview` — 仪表盘统计数据
- `GET /participation` — 参与矩阵
- `GET /user/:id` — 个人参与历史

## 七、数据模型

```
users          papers            comments         likes
─────          ──────            ────────         ─────
id (PK)        id (PK)           id (PK)          id (PK)
username       title             paper_id (FK)    comment_id (FK)
email          authors           user_id (FK)     user_id (FK)
password_hash  abstract          parent_id (FK)   created_at
real_name      file_path         content
student_id     file_size         created_at       UNIQUE(comment_id,
research_dir   original_fname    updated_at              user_id)
role           uploader_id (FK)
is_active      present_date
created_at     created_at
updated_at     updated_at
```

**关系**：
- User 1:N Paper（上传关系）
- User 1:N Comment（评论关系）
- Paper 1:N Comment
- Comment 1:N Comment（自引用，回复嵌套）
- Comment 1:N Like
- User 1:N Like（UNIQUE 约束防重复点赞）
